## 本次新增试验

### Telecommunicating

- range: pp.138-157
- anchor: 1985 年的电脑通信不是外设功能，而是个人计算机的社会未来。
- asset: `/assets/wholeearth_1985_visuals/siamese-cat-telecommunicating-v1.png`
- judgement: 成功。猫的角色稳定，电话线、邮件、公告板、文件传输、社群这些对象能独立于 Micro-Prolog 内容成立。

### Learning

- range: pp.175-191
- anchor: 好的学习软件不是电子练习册，而是可试错、可操纵、可想象的微世界。
- asset: `/assets/wholeearth_1985_visuals/siamese-cat-learning-v1.png`
- judgement: 概念成功，但画面文字偏多。正式版应压缩为“练习册 / 微世界 / 试错 / 反馈”四个核心标签。

### Organizing

- range: pp.78-93
- anchor: 数据库不是存东西的箱子，而是设计未来找回方式的工具。
- asset: `/assets/wholeearth_1985_visuals/siamese-cat-organizing-v1.png`
- judgement: 最成功。左右对照天然清楚：从被数据淹没，到可设计地找回。适合做成正式页面入口。

## 后续打磨优先级

1. **Shopping / pp.4-9**：不要先问哪个软件最火，先定义流程、指标、限制和支持。
2. **Hardware / pp.14-21**：买硬件不是买机器，而是进入或放弃一个软件世界。
3. **Writing / pp.46-63**：好的文字处理器改变的是修改、重排和思考文本的方式；“不要买的理由”比功能表更诚实。
4. **Analyzing / pp.64-77**：电子表格让非程序员开始建模未来，也带来可读性和错误控制问题。
5. **Managing / pp.106-121**：一体化平台和小工具生态的路线竞争，延续到今天。
6. **Update / pp.201-224**：最好的技术未必赢，软件生态、兼容性和开发者承诺会决定标准。

## 使用边界

- Programming 章已有逐页样板，可以直接做页级插画。
- 其他章节当前只能算章节级锚点；正式发布前需要回到扫描图补关键页或跨页证据。
- 暹罗猫形象可以固定，章节内容不能固定。每张图都应重新生成视觉隐喻和标签。
