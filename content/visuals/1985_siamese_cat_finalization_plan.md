# 1985 Software Catalog 暹罗猫插画终稿推进计划

状态：内部制作文件，不面向读者。目标是把现有 17 张章节级视觉勘探稿，推进为可发布的读者小册子。

## 终稿标准

每张终稿必须满足：

1. 有明确章节范围和扫描页范围。
2. 至少锚定一个具体对象：页面判断、软件类别、比较表、书评、工具链、使用场景或章节开头判断。
3. 图像不是复用通用模板，而是根据该章材料重新设计隐喻。
4. 图内文字控制在 4-6 个短标签，不放长段解释。
5. 读者文案只解释内容，不出现“候选、草稿、正式版、需要重跑”等制作话术。
6. 图像旁保留页码范围；更精确的页级专题必须保留具体 leaf/page 和扫描链接。

### Organizing / pp.78-93

- source range: scanned leaves 80-95
- source object: 数据库工具从卡片盒、字段、报表走向 dBASE/R:Base 这类半编程环境。
- anchor: 数据库不是存东西的箱子，而是设计未来找回方式的工具。
- final metaphor: 左侧信息堆把人淹没，右侧通过命名、字段、关系、查询路径把同一批资料变成可找回结构。
- labels: 命名, 字段, 关系, 查询, 找回

### Telecommunicating / pp.138-157

- source range: scanned leaves 140-159
- source object: 调制解调器、通信软件、在线服务、公告板、电子邮件、会议系统和文件传输共同构成早期网络社会。
- anchor: 1985 年的电脑通信不是外设功能，而是个人计算机的社会未来。
- final metaphor: 电话线接出的社会地图；终端不是中心，邮件、公告板、文件传输和社群才是中心。
- labels: 调制解调器, 邮件, 公告板, 文件, 社群

### Managing / pp.106-121

- source range: scanned leaves 108-123
- source object: AppleWorks、Framework、Symphony 等集成软件，与 SideKick、Switcher 等小工具路线并存。
- anchor: 现代办公软件的两条路线早已出现：一体化平台，或围绕工作流的小工具生态。
- final metaphor: 一张办公桌被分成两套工具路线：一体化工作台和可弹出的小工具环。
- labels: 一体化, 小工具, 数据流, 切换, 流程

### Hardware / pp.14-21

- source range: scanned leaves 16-23
- source object: Apple II、Commodore 64、IBM PC、Macintosh、AT&T UNIX PC 等机器被放在软件生态、兼容性和未来风险里比较。
- anchor: 买硬件不是买参数，而是在选择能进入哪个软件世界。
- final metaphor: 机器不是商品盒，而是通向不同软件库的门；有的门厚、有的门新、有的门通向程序员工作坊。
- labels: 机器, 软件库, 兼容, 支持, 风险

### Learning / pp.175-191

- source range: scanned leaves 177-193
- source object: Logo、Robot Odyssey、Rocky's Boots 等让学习者进入可试错、可模拟、可操纵的系统。
- anchor: 好的学习软件不是电子练习册，而是可试错、可操纵、可想象的微世界。
- final metaphor: 练习册托盘与微世界托盘对照；微世界里有规则、反馈、路径和可修改对象。
- labels: 练习册, 微世界, 试错, 反馈, 想象

### Programming / pp.158-174

- source range: scanned leaves 159-176
- source object: 从 Weinberg 的软件素养开篇，到程序风格、第二语言、UNIX、结构化设计、软件工程和 utilities。
- anchor: 编程素养不是职业专利，而是理解软件黑箱、判断工具、重新获得控制权的方法。
- final metaphor: 普通用户打开软件黑箱，里面不是一门语言，而是一组风格、设计、环境、工程和修复工具。
- labels: 黑箱, 风格, 设计, 环境, 控制权
