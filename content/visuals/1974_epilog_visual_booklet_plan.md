## 方向修正

本册不做封面 spread，不做使用说明 spread，也不做项目介绍页。册子直接进入《Whole Earth Epilog, October 1974》的十个原书章节，每章用具体内容、条目群和可视化锚点来呈现。

章节标题可以保留原书结构，但读者页标题要更具体。比如 `土地使用` 在原书中不是土地开发，而是自然史、蘑菇、昆虫、植物、水、厕所、污水、土壤、小农、牲畜、园艺和室内植物共同组成的生命循环章节。读者页应写成 `土地不是空地：蘑菇、水、粪便和土壤循环` 这类具体标题。

## 小册子结构

章节是组织单位，不是固定的视觉页单位。每个原书章节可以拆成多个 booklet modules；module 才是具体图像、页面或短小册 spread 的单位。

第一版先按 10 个原书章节组织：

1. 整体系统：工具进入系统以后会发生什么
2. 土地使用：土地不是空地，生命循环才是主角
3. 住所：房子要能被地点、材料和失败纠正
4. 软技术：替代不是口号，能交付才算数
5. 手艺：身体保存的知识
6. 共同体：愿望必须长出基础设施
7. 游牧：移动能力来自地图、维修和风险意识
8. 通信：谁能说、听、复制、查找、修理媒介
9. 学习：没有错误，只有制作
10. 出版业务、索引与封底：一本书公开自己的生产系统

每章至少包含：

- 一个读者标题
- 按完整中文章稿拆出的全部必要 module
- 每个 module 的具体内容点
- 每个 module 的视觉锚点
- 每个 module 的 4 到 6 个短标签
- 每个 module 的一句读者说明

模块数量不预设上限。如果某章内容特别密集，就继续拆到内容结构被表达清楚为止。不要为了形式整齐强行“一章一图”，也不要为了省图把完整中文章稿压缩成少数抽象主题。

所有 module 必须先从完整中文章稿中抽取，不从标题、页码证据、OCR 英文条目或视觉想象反推内容。页码、leaf 和 scan 只做证据定位，不做内容来源替代。

## 视觉原则

沿用 `skills/siamese-cat-knowledge-illustrations/SKILL.md` 的暹罗猫阅读室编辑形象，但角色只作为引导者，不作为主题。

1974 这本应比 1985 软件小册子更像“档案工具台”：

- 暖白纸面
- 手绘线稿和少量排线
- 棕黑墨线、旧纸色、档案蓝、少量警示红
- 页边索引、卡片、线、章签、邮寄标签、价目表、地图格线
- 每张图只放 4 到 6 个中文短标签

避免：

- 把每章画成同一种机器
- 把 Whole Earth 画成怀旧嬉皮 collage
- 把工具观简化为现代效率或生产力话术
- 用封面、导语、项目说明挤占章节内容

## Chapter Module 设计表

### 01. 整体系统：工具进入系统以后会发生什么

- 来源章节：`一、整体系统`
- 具体内容：Bateson 的反馈与元层级；Buber 的关系伦理；Sagan 与地外视角；《增长的极限》；Illich 的能源与公平；Odum 的能源、生态与经济。
- 候选 module：
  - `反馈与元层级`：Bateson、控制论、回路、语境、悖论。标签：`反馈`、`回路`、`语境`、`悖论`、`系统心智`。
  - `关系不是控制`：Buber 的“我-你”、关系伦理、共同进化。标签：`我-你`、`关系`、`不占有`、`共同进化`。
  - `从太空和身体重新看地球`：Sagan、地外文明、卫星照片、身体内部图像。标签：`地外视角`、`卫星影像`、`身体宇宙`、`重新估计人类`。
  - `增长模型与能源政治`：《增长的极限》、Illich、核能、Odum。标签：`增长`、`速度`、`核能`、`净能量`、`崩溃延迟`。
- 章节读者句：这一章把后面所有工具都放回系统里看：工具可能扩大人的能力，也可能让人变成巨型制度的看护者。

  - `蘑菇：耐心、风险与拉丁名`：多本指南、孢子印、拉丁名、危险食物、互证。标签：`蘑菇`、`孢子印`、`拉丁名`、`耐心`、`风险`。
  - `昆虫、蜘蛛与鸟类`：昆虫武器、蜘蛛指南、技术检索表、鸟类指南服务现场观察。标签：`昆虫`、`蜘蛛`、`鸟`、`观察`、`指南`。

  - `植物、草药与树`：植物生物学、仙人掌、植物术语、草药、药物来源、植树恢复、树木作物。标签：`植物词汇`、`草药`、`药物`、`树`、`恢复`。

- 章节读者句：这一章从识别蘑菇和鸟开始，最后落到厕所、污水和土壤，因为土地不是背景，而是许多生命共同维持的系统。

## 整本开发顺序

按原书章节顺序推进，但每章按完整中文章稿的内容密度拆成多个 module。模块总数不固定，随着逐章重读完整中文稿更新。当前 `土地使用` 已重拆为 11 个 module，旧的 4-module 方案作废。

1. 整体系统：4 个 module
2. 土地使用：11 个 module
3. 住所：4 个 module
4. 软技术：4 个 module
5. 手艺：3 个 module
6. 共同体：4 个 module
7. 游牧：4 个 module
8. 通信：4 个 module
9. 学习：4 个 module
10. 出版业务、索引与封底：3 个 module

开发顺序：

1. 先完成整本 reader-facing 文案。
2. 再为每个完整 module 建立 leaf / page / scan 证据索引。
3. 再逐 module 展开插画 prompt 变量。
4. 最后生成图像、插入 HTML，并做版面 QA。

任何阶段都不要退回“一章一图”的压缩逻辑。
